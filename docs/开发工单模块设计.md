# 开发工单模块设计

## 1. 状态流转

六阶段生命周期：

```
开发单已创建 (created)
    │
    ▼
设计方案已评审 (design_reviewed)
    │
    ▼
开发已完成 (dev_completed)
    │
    ▼
测试已完成 (test_completed)
    │
    ▼
已上线 (live)
    │
    ▼
已归档 (archived)
```

## 2. 数据表设计

### 2.1 dev_ticket — 开发工单主表

```sql
CREATE TABLE pmwb_dev_ticket (
    id INT AUTO_INCREMENT PRIMARY KEY COMMENT '自增ID',

    -- 工单编号（各系统共用同一开发单号）
    ticket_no VARCHAR(64) NOT NULL COMMENT '开发工单编号 (如 DTS-2026-xxxx)',
    req_id VARCHAR(64) NOT NULL COMMENT '关联需求编号 (对应 sent_emails.req_id)',

    -- 业务归属（同一需求下，不同系统的工单以 system_name 区分）
    system_name VARCHAR(128) NOT NULL COMMENT '涉及系统 (CRM/BOSS/HDC/订单中心等)',
    dev_team VARCHAR(64) COMMENT '开发团队/厂商',
    developer VARCHAR(64) COMMENT '开发负责人',
    dev_contact VARCHAR(128) COMMENT '开发联系方式',

    -- 时间线
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '工单创建时间',
    design_reviewed_date DATE COMMENT '设计方案评审日期',
    dev_completed_date DATE COMMENT '实际完成开发日期',
    test_completed_date DATE COMMENT '测试完成日期',
    go_live_date DATE COMMENT '实际上线日期',
    archived_date DATE COMMENT '归档日期',

    -- 状态
    status ENUM('created','design_reviewed','dev_completed','test_completed','live','archived')
           DEFAULT 'created' COMMENT '当前状态',
    progress TINYINT DEFAULT 0 COMMENT '进度百分比 0-100',

    -- 详情
    description TEXT COMMENT '工单描述/开发内容',
    risk_note TEXT COMMENT '风险/延期原因',

    -- 交付物
    deliverable_path VARCHAR(512) COMMENT 'Obsidian交付物归档路径',

    -- 标记
    is_overdue TINYINT(1) DEFAULT 0 COMMENT '是否超期',
    priority ENUM('P0','P1','P2','P3') DEFAULT 'P2' COMMENT '优先级',

    -- 元数据
    created_by VARCHAR(64) COMMENT '创建人',
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

    -- 联合唯一约束：同一开发单号 + 不同系统
    UNIQUE KEY uk_ticket_system (ticket_no, system_name),
    INDEX idx_req_id (req_id),
    INDEX idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='开发工单表';
```

### 2.2 pmwb_dev_deliverable — 交付物记录表

```sql
CREATE TABLE pmwb_dev_deliverable (
    id INT AUTO_INCREMENT PRIMARY KEY COMMENT '自增ID',
    ticket_id INT NOT NULL COMMENT '关联 pmwb_dev_ticket.id',

    -- 交付物类型（操作手册按文件名关键词自动识别）
    deliverable_type ENUM('operation_manual','interface_doc','test_case','release_note','other')
                     NOT NULL DEFAULT 'other' COMMENT '交付物类型',
    file_name VARCHAR(255) NOT NULL COMMENT '文件名',
    original_name VARCHAR(255) COMMENT '原始文件名',
    file_size INT COMMENT '文件大小(字节)',
    file_type VARCHAR(64) COMMENT '文件MIME类型',

    -- 存储位置
    obsidian_path VARCHAR(512) COMMENT 'Obsidian归档完整路径',
    local_path VARCHAR(512) COMMENT '本地备份路径',

    -- 来源
    source VARCHAR(32) DEFAULT 'upload' COMMENT '来源 (plugin/upload)',
    source_url VARCHAR(1024) COMMENT '来源URL（插件捕获时记录）',
    note TEXT COMMENT '备注',

    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,

    INDEX idx_ticket_id (ticket_id),
    FOREIGN KEY (ticket_id) REFERENCES pmwb_dev_ticket(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='开发交付物表';
```

### 2.3 pmwb_dev_ticket_log — 状态变更日志

```sql
CREATE TABLE pmwb_dev_ticket_log (
    id INT AUTO_INCREMENT PRIMARY KEY,
    ticket_id INT NOT NULL COMMENT '关联 pmwb_dev_ticket.id',
    from_status VARCHAR(32) COMMENT '变更前状态',
    to_status VARCHAR(32) COMMENT '变更后状态',
    operator VARCHAR(64) COMMENT '操作人',
    note TEXT COMMENT '变更原因/备注',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,

    INDEX idx_ticket_id (ticket_id),
    FOREIGN KEY (ticket_id) REFERENCES pmwb_dev_ticket(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='开发工单状态变更日志';
```

```
│ background            │           │ progress (0-100%)    │
│ description           │           │ design_reviewed_date  │
│ clarification         │           │ dev_completed_date    │
│ propose_time          │           │ test_completed_date   │
│ send_datetime         │           │ go_live_date          │
│ created_at            │           │ archived_date         │
└──────────────────────┘           │ deliverable_path      │
                                    │ priority              │
                                    │ is_overdue            │
                                    └───────────────────────┘
                                               │
                                               ▼
                                    ┌──────────────────────┐
                                    │ pmwb_dev_deliverable  │
                                    │ (交付物)               │
                                    ├──────────────────────┤

                                    └──────────────────────┘

唯一性约束: UNIQUE(ticket_no, system_name)
```

## 4. 交付物归档到Obsidian

### 归档路径

```
02-需求资产/
├── 交付物/
│   ├── {req_id}/
│   │   ├── {ticket_no}/
│   │   │   ├── {操作手册名}.docx
│   │   │   ├── {接口文档}.docx
│   │   │   ├── {测试用例}.xlsx
│   │   │   └── 交付物清单.md       ← 自动生成
│   │   └── (多个团队并存)
│   │       ├── CRM/
│   │       └── BOSS/
│   └── ...
├── 需求卡片/     ← 保持不动
└── 操作手册/     ← 保持不动
```

### 文件名识别规则

文件名包含"操作手册" → `deliverable_type = 'operation_manual'`
（其他类型按需后续扩展）

## 5. 采集方案

- **插件端**：从内网开发工单系统页面对接，提取工单编号、状态、负责人等，后续逐步细化
- **Web端**：产品经理个人工作台提供手动创建/编辑开发工单功能

## 6. 交付物归档方式

- **浏览器插件自动捕获**：用户在工单系统页面下载文件时触发
- **Web端手动上传**：在工作台页面选择文件上传
- 两种方式均支持，最终归档路径统一
